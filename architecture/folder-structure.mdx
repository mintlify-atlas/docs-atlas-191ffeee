## Folder descriptions

<AccordionGroup>
  <Accordion title="components/">
    Shared, reusable UI components that are not tied to a single page. This includes the application layout primitives:

    - `layout/PrivateLayout.tsx` ‚Äî wrapper rendered around every authenticated page; mounts the `Sidebar` and `BottomNav`.
    - `layout/Sidebar.tsx` ‚Äî collapsible desktop navigation.
    - `layout/BottomNav.tsx` ‚Äî mobile tab bar.
    - `ErrorBoundary.tsx` ‚Äî class component that catches render errors and shows a recovery UI.

    If a component is used by more than one page, it belongs here. If it is used by exactly one page, co-locate it with that page instead.
  </Accordion>

  <Accordion title="pages/">
    One subfolder per application domain. Each subfolder contains:

    - A top-level page component (`DashboardPage.tsx`)
    - A co-located styles file (`DashboardPage.styles.ts`)
    - An optional `components/` subfolder for modals, cards, and list items that belong exclusively to that page

    | Subfolder | Contents |
    |---|---|
    | `auth/` | `LoginPage`, `SignupPage` |
    | `challenges/` | Catalogue, active challenge card, create/abandon/edit modals |
    | `tasks/` | Full CRUD task list and task item component |
    | `dashboard/` | Gamified progress overview with XP and streak metrics |
    | `home/` | Public landing page (`/`) |
    | `profile/` | Profile view and password change |
  </Accordion>

  <Accordion title="services/">
    The REST communication layer. Each file is a plain object whose methods call the shared Axios instance and return typed responses:

    - `api.ts` ‚Äî the configured Axios instance with JWT and 401 interceptors
    - `auth.service.ts` ‚Äî login and signup
    - `challenge.service.ts` ‚Äî challenge templates and user challenges
    - `task.service.ts` ‚Äî task CRUD and completions
    - `user.service.ts` ‚Äî profile read and update

    Service files contain no UI logic. They are the only layer that imports from `api.ts`.
  </Accordion>

  <Accordion title="store/">
    Zustand stores for global state:

    - `authStore.ts` ‚Äî authentication session (user, token, `isAuthenticated`)
    - `themeStore.ts` ‚Äî current theme preference (`isDark`)
    - `appCache.ts` ‚Äî in-memory cache for dashboard metrics, challenges, tasks, and profile data
  </Accordion>

  <Accordion title="styles/">
    Global styling primitives:

    - `theme.ts` ‚Äî `darkTheme`, `lightTheme`, and the `Theme` type export
    - `GlobalStyle.ts` ‚Äî CSS reset and base body/font styles via `createGlobalStyle`
  </Accordion>

  <Accordion title="types/">
    TypeScript interfaces and type aliases for API DTOs:

    - `auth.types.ts`
    - `challenge.types.ts`
    - `task.types.ts`
    - `user.types.ts`
    - `styled.d.ts` ‚Äî module augmentation that extends styled-components' `DefaultTheme` with the project's `Theme` type
  </Accordion>

  <Accordion title="routes/">
    - `AppRoutes.tsx` ‚Äî all route definitions, `PrivateRoute`, and `PublicRoute` guard components
  </Accordion>
</AccordionGroup>

</Tip>
