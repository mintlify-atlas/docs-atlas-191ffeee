## Theme type

The `Theme` type is derived directly from `darkTheme` to keep the type and the runtime object in sync:

```typescript
export type Theme = typeof darkTheme;
```

This means any new token you add to `darkTheme` is automatically reflected in the type. You do not maintain a separate interface.

## ThemeProvider setup

`main.tsx` reads the current preference from `useThemeStore` and passes the appropriate theme object to styled-components' `ThemeProvider`:

```tsx
// src/main.tsx
const Root = () => {
  const isDark = useThemeStore((s) => s.isDark);
  const theme = isDark ? darkTheme : lightTheme;

  return (
    <ThemeProvider theme={theme}>
      <GlobalStyle />
      <App />
      <Toaster
        toastOptions={{
          style: {
            background: theme.colors.surface,
            color: theme.colors.textPrimary,
            border: `1px solid ${theme.colors.border}`,
          },
        }}
      />
    </ThemeProvider>
  );
};
```

When the user calls `toggleTheme()`, the store updates `isDark`, `Root` re-renders with the new theme object, and all styled components instantly reflect the new token values.

<Tip>
  The app defaults to dark mode. If `localStorage('theme')` is absent or any value other than `'light'`, the dark theme is active.
</Tip>
