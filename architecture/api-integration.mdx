All HTTP communication goes through a single configured Axios instance defined in `src/services/api.ts`. Service files import this instance and never call Axios directly.

## Axios instance

```typescript
// src/services/api.ts
import axios from "axios";

if (!import.meta.env.VITE_API_URL) {
  throw new Error("VITE_API_URL is not defined in .env");
}

const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL,
});
```

<Warning>
  The application throws at startup if `VITE_API_URL` is not set. Make sure your `.env` file contains this variable before running the dev server.
</Warning>

## Environment variable

Create a `.env` file at the project root:

```bash
VITE_API_URL=http://localhost:3000/api
```

Vite exposes all variables prefixed with `VITE_` to the client bundle via `import.meta.env`.

## JWT request interceptor

Every outgoing request automatically gets an `Authorization: Bearer <token>` header if a token exists in `localStorage`:

```typescript
// Request interceptor
api.interceptors.request.use(
  (config) => {
    const token = localStorage.getItem("token");
    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
  },
  (error) => Promise.reject(error),
);
```

The interceptor reads the token from `localStorage` on every request so it always uses the most current value without needing a store subscription.

## 401 response interceptor

When the API returns a `401 Unauthorized` response, the interceptor clears the stored token and redirects the user to the home page — unless the current page is already an auth page, in which case it lets the page's own error handling take over:

```typescript
// Response interceptor
api.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      const path = window.location.pathname;
      // Don't redirect on auth pages — let them show the error message
      const isAuthPage = path === "/login" || path === "/signup" || path === "/";
      if (!isAuthPage) {
        localStorage.removeItem("token");
        sessionStorage.removeItem("_up");
        window.location.href = "/";
      }
    }
    return Promise.reject(error);
  },
);
```

<Info>
  The response interceptor uses `window.location.href` for the redirect rather than React Router's `navigate()`. This performs a full page reload, which also resets all Zustand store state — effectively a clean logout.
</Info>

## Service layer pattern

Each domain has its own service file. A service is a plain object whose methods call `api` and return typed responses:

```typescript
// Pattern: call api, return response.data cast to the expected type
export const authService = {
  login: async (email: string, password: string) => {
    const response = await api.post(`/auth/login`, { email, password });
    return response.data as AuthResponse;
  },
};
```
